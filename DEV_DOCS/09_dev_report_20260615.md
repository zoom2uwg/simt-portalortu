## 🔧 Rincian Pembaruan File Deployment Terakhir

Dalam pembaruan terakhir, file-file deployment yang sebelumnya di-copy dari template lama (`simt-mts`) telah diperbaiki agar berfungsi secara out-of-the-box di server target tanpa modifikasi manual:

### 1. `start.sh` (Shell Script Auto-Restart)
* **Sebelumnya (Hardcoded):**
  ```bash
  cd /home/z/my-project
  echo "Starting SIMT MTs server with auto-restart..."
  ```
* **Sesudahnya (Dinamis & Spesifik):**
  ```bash
  cd "$(dirname "$0")"
  echo "Starting SIMT Portal Ortu server with auto-restart..."
  ```
* **Benefit:** Script otomatis masuk ke direktori tempat ia diletakkan, tidak peduli apakah diletakkan di `/var/www/simt-portalortu` or `/home/ubuntu/portalortu`. Nama log juga sudah disesuaikan.

### 2. `ecosystem.config.js` (PM2 Process Manager)
* **Sebelumnya (Hardcoded):**
  ```javascript
  name: 'simt-mts',
  script: 'node_modules/.bin/next',
  cwd: '/home/z/my-project',
  ```
* **Sesudahnya (Dinamis & Spesifik):**
  ```javascript
  name: 'simt-portalortu',
  script: 'node_modules/next/dist/bin/next',
  cwd: __dirname,
  ```
* **Benefit:** 
  * Nama proses PM2 menjadi `simt-portalortu` sehingga tidak bentrok dengan aplikasi SIMT MTs lainnya di server yang sama.
  * Direktori kerja (`cwd`) diset dinamis ke directory file ini berada (`__dirname`).
  * Menggunakan binary Next.js langsung (`node_modules/next/dist/bin/next`) dibanding wrapper `.bin/next` agar PM2 dapat mengontrol, me-restart, dan mencatat log proses Node dengan sempurna tanpa proses wrapper tambahan.

### 3. `Caddyfile` (Web Server & Reverse Proxy)
* **Status:** Generic & Valid
* **Fungsi:** Mengatur traffic pada port `:81` ke port `:3000` (atau ke port custom menggunakan query string `XTransformPort` untuk keperluan testing multi-tenant).
* **Konfigurasi Aktif:**
  ```caddyfile
  :81 {
      # Custom header proxy routing
      @transform_port_query {
          query XTransformPort=*
      }
      handle @transform_port_query {
          reverse_proxy localhost:{query.XTransformPort} { ... }
      }
      # Default routing
      handle {
          reverse_proxy localhost:3000 { ... }
      }
  }
  ```

## 📋 Langkah Pengujian & Deployment Selanjutnya

Untuk menjalankan aplikasi ini di server VPS menggunakan konfigurasi yang baru diperbarui:

### 1. Inisialisasi Project di VPS:
```bash
# Clone dan masuk ke folder project
git clone <repository_url> /var/www/simt-portalortu
cd /var/www/simt-portalortu

# Copy environment variables dan sesuaikan isinya
cp .env.example .env
nano .env

# Install dependencies (menggunakan Bun)
bun install
```

### 2. Jalankan Build & Database Migration:
```bash
# Generate Prisma Client & Build Next.js
bun run build

# Push skema database (jika menggunakan database baru)
bun run db:push
```

### 3. Menjalankan Aplikasi di Background (Pilih Salah Satu):

* **Opsi A: Menggunakan PM2 (Rekomendasi Utama)**
  ```bash
  # Jalankan via PM2 menggunakan config ecosystem.js yang sudah diperbaiki
  pm2 start ecosystem.config.js
  
  # Simpan proses agar berjalan otomatis setelah restart server
  pm2 save
  pm2 startup
  ```

* **Opsi B: Menggunakan Script Shell Autorestart**
  ```bash
  # Berikan izin eksekusi ke script
  chmod +x start.sh
  
  # Jalankan di background menggunakan nohup/screen
  nohup ./start.sh > server.log 2>&1 &
  ```

### 4. Konfigurasi Caddy Server:
Salin konfigurasi Caddyfile ke Caddy system directory atau muat ulang Caddy dengan konfigurasi lokal:
```bash
caddy reload --config Caddyfile
```
